## 1. Что это за инструмент

**AI Analytics Agent** — это веб-приложение для анализа прибыльности IT-проектов. Оно автоматически:

- **Собирает данные** из базы данных или CSV-файлов (проекты, разработчики, задачи, финансы)
- **Рассчитывает метрики** прибыльности по проектам, технологическим стекам, типам задач и разработчикам
- **Выявляет аномалии** — убыточные проекты, перерасход часов, частые переоткрытия задач
- **Генерирует AI-гипотезы** о причинах потерь и рекомендации по оптимизации
- **Строит интерактивные графики** — маржинальность, burn rate, сравнение стеков
- **Подсвечивает цветом** — 🟢 прибыльные (маржа >15%), 🟡 пограничные (0-15%), 🔴 убыточные (<0%)

### Ключевые метрики

| Метрика | Формула | Описание |
|---------|---------|----------|
| **Profitability (маржа)** | `(доход - расходы) / доход × 100%` | Процент прибыли от дохода |
| **Burn Rate** | `расходы / кол-во месяцев` | Средний расход в месяц |
| **Revenue/Hour** | `доход / факт. часы` | Доход на час работы |
| **Overtime Ratio** | `факт. часы / оценочные часы` | Коэффициент перерасхода времени |
| **Reopen Rate** | `переоткрытия / общее кол-во задач` | Доля задач с переоткрытиями |
| **Estimation Accuracy** | `оценочные часы / факт. часы` | Точность оценки задач |

## 3. Установка и первый запуск

### Шаг 1: Откройте терминал (командную строку)

На Windows: нажмите `Win + R`, введите `cmd` или `powershell`, нажмите Enter.

### Шаг 2: Перейдите в папку проекта

```bash
cd "путь/к/папке/AI_ANALYTICS_AGENT"
```

### Шаг 3: Установите зависимости

```bash
pip install -r requirements.txt
```

> **Примечание:** Если команда `pip` не найдена, попробуйте `python -m pip install -r requirements.txt`

### Шаг 4: Запустите приложение

```bash
python main.py
```

Вы увидите примерно такой вывод:

```

  Extracted developers: 30 records
```

## 4. Страницы дашборда

### 📊 Главный дашборд (`/`)

Основная страница с:
- **KPI-карточками** — общий доход, расходы, прибыль, маржа, кол-во убыточных проектов
- **Графики** — маржинальность проектов (горизонтальный bar chart), соотношение прибыльных/убыточных (pie chart), доход vs расходы по стекам, burn rate по проектам
- **AI Аналитика** — гипотезы AI о причинах потерь (с уверенностью в %) и рекомендации по оптимизации
- **Аномалии** — список выявленных проблем, отсортированных по критичности

### 📁 Проекты (`/projects`)

Детальная таблица по всем проектам:
- Название, стек, доход, расходы, прибыль, маржа, burn rate, $/час
- Цветовой статус: 🟢 Прибыльный / 🟡 Пограничный / 🔴 Убыточный
- График маржинальности

### ⚡ Стеки (`/stacks`)

Сравнение технологий (Unity, UE, Flutter, Vue.js, Laravel, ASP.NET):
- KPI-карточка для каждого стека с маржой и цветом
- Таблица: доход, расходы, прибыль, маржа, $/час, часы
- График сравнения доходов и расходов

### 📋 Задачи (`/tasks`)

Анализ по типам задач (feature, bug, improvement):
- Перерасход часов (overtime ratio)
- Стоимость по типам
- Reopen rate
- Список аномальных задач (перерасход >2x, переоткрытия >3)

### 👥 Разработчики (`/developers`)

Метрики эффективности разработчиков:
- Часы, стоимость, кол-во задач
- Точность оценок, перерасход, скорость
- Шкала эффективности (0-100)

## 5. API-эндпоинты

Система предоставляет REST API для интеграции с другими инструментами:

| Метод | URL | Описание |
|-------|-----|----------|
| `GET` | `/api/summary` | Полные данные дашборда в JSON |
| `GET` | `/api/anomalies` | Список обнаруженных аномалий |
| `GET` | `/api/ai-analysis` | AI-анализ: гипотезы + рекомендации |
| `POST` | `/api/etl/run` | Вручную запустить ETL-пайплайн |

### Пример использования API:

```bash
# Получить сводку
curl http://localhost:8000/api/summary

# Получить аномалии
curl http://localhost:8000/api/anomalies

# Перезапустить ETL
curl -X POST http://localhost:8000/api/etl/run
```

## 6. Подключение реальных данных

По умолчанию система работает с **демо-данными** (18 проектов, 30 разработчиков, 73 задачи). Чтобы подключить свои данные:

### Вариант A: Через CSV-файлы (самый простой)

1. Подготовьте 4 CSV-файла в формате, аналогичном файлам в `data/demo/`:
   - `projects.csv` — проекты
   - `developers.csv` — разработчики
   - `tasks.csv` — задачи
   - `financials.csv` — финансовые данные

2. Положите их в папку (например, `data/my_data/`)

3. Измените файл `.env`:
   ```
   DATA_MODE=demo
   ```
   И в файле `etl/extract.py` укажите путь к вашим данным, или создайте свой `DemoExtractor` с нужным `data_dir`.

### Вариант B: Через MS SQL Server

1. Убедитесь, что установлен **ODBC Driver for SQL Server**:
   - Скачайте с сайта Microsoft: https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server
   - Установите драйвер
   - Установите Python-пакет: `pip install pyodbc`

2. Отредактируйте файл `.env`:
   ```env
   DATA_MODE=mssql
   MSSQL_HOST=ваш_сервер
   MSSQL_PORT=1433
   MSSQL_DATABASE=имя_базы
   MSSQL_USER=логин
   MSSQL_PASSWORD=пароль
   MSSQL_DRIVER=ODBC Driver 17 for SQL Server
   ```

3. Убедитесь, что в базе есть таблицы `projects`, `developers`, `tasks`, `financials` с полями, соответствующими CSV-файлам.

4. Перезапустите приложение.

### Формат CSV-файлов

#### projects.csv
```
id,name,project_type,client,stack,status,start_date,end_date
1,Мой Проект,web,Клиент,Vue.js,active,2025-01-01,
```

#### developers.csv
```
id,name,role,stack,hourly_rate,monthly_salary
1,Иван Иванов,senior,Vue.js,40,6400
```

#### tasks.csv
```
id,external_id,title,task_type,status,priority,project_id,developer_id,estimated_hours,actual_hours,reopen_count,comments_count,created_date,completed_date
1,TASK-1,Описание задачи,feature,done,high,1,1,20,18,0,5,2025-01-01,2025-01-15
```

#### financials.csv
```
id,project_id,period,revenue,costs,labor_costs,overhead_costs
1,1,2025-01-01,50000,35000,28000,7000
```

## 7. Подключение AI (OpenAI / Claude)

По умолчанию система использует **правила (rule-based анализ)** — она выявляет паттерны без AI. Для подключения полноценного AI-анализа:

### Вариант A: OpenAI (GPT-4o)

1. Получите API-ключ на https://platform.openai.com/api-keys
2. Установите библиотеку: `pip install openai`
3. Отредактируйте `.env`:
   ```env
   AI_PROVIDER=openai
   OPENAI_API_KEY=sk-ваш-ключ
   OPENAI_MODEL=gpt-4o
   ```

### Вариант B: Anthropic (Claude)

1. Получите API-ключ на https://console.anthropic.com/
2. Установите библиотеку: `pip install anthropic`
3. Отредактируйте `.env`:
   ```env
   AI_PROVIDER=anthropic
   ANTHROPIC_API_KEY=sk-ant-ваш-ключ
   ANTHROPIC_MODEL=claude-sonnet-4-20250514
   ```

4. Перезапустите приложение.

> **Примечание:** AI будет получать контекст с финансовыми данными и аномалиями, затем генерировать гипотезы и рекомендации на русском языке в структурированном формате.

## 8. Подключение Google Sheets

Для автоматического экспорта данных в Google Sheets:

### Шаг 1: Создайте сервисный аккаунт Google

1. Откройте https://console.cloud.google.com/
2. Создайте проект (или выберите существующий)
3. Включите **Google Sheets API** и **Google Drive API**
4. Перейдите в **APIs & Services → Credentials**
5. Нажмите **Create Credentials → Service Account**
6. Создайте ключ JSON и скачайте файл

### Шаг 2: Настройте проект

1. Сохраните JSON-файл в папку `credentials/google_service_account.json`
2. Отредактируйте `.env`:
   ```env
   GOOGLE_SHEETS_ENABLED=true
   GOOGLE_SERVICE_ACCOUNT_FILE=credentials/google_service_account.json
   GOOGLE_SPREADSHEET_NAME=AI Analytics Agent Report
   ```

3. Перезапустите приложение.

> Система автоматически создаст таблицу Google Sheets с листами «Проекты» и «Стеки» и заполнит данными.

## 9. Автоматический запуск ETL по расписанию

Чтобы ETL-пайплайн запускался автоматически (например, каждые 24 часа):

1. Отредактируйте `.env`:
   ```env
   ETL_SCHEDULE_ENABLED=true
   ETL_SCHEDULE_HOURS=24
   ```

2. Перезапустите приложение. В логах вы увидите:
   ```
   Scheduler started: ETL every 24h
   ```

Можно также запустить ETL вручную через API:
```bash
curl -X POST http://localhost:8000/api/etl/run
```

## 13. Часто задаваемые вопросы

### Вопрос: Приложение не запускается, ошибка «ModuleNotFoundError»
**Ответ:** Установите зависимости: `pip install -r requirements.txt`

### Вопрос: Порт 8000 уже занят
**Ответ:** Измените порт в файле `.env`:
```env
APP_PORT=8080
```

### Вопрос: Как изменить язык интерфейса?
**Ответ:** В файле `.env` измените `UI_LANGUAGE=en` (сейчас поддерживается только русский)

### Вопрос: Можно ли использовать без AI (OpenAI/Claude)?
**Ответ:** Да! По умолчанию `AI_PROVIDER=none`, и система использует правила для анализа. Она всё равно выявляет убыточные проекты, аномалии и генерирует гипотезы на основе статистики.

### Вопрос: Как часто обновляются данные?
**Ответ:** По умолчанию — при каждом запуске приложения. Для автообновления включите планировщик (`ETL_SCHEDULE_ENABLED=true`). Можно также обновить вручную через `POST /api/etl/run`.

### Вопрос: Где хранятся аналитические данные?
**Ответ:** В файле `data/analytics.db` (SQLite). Этот файл создаётся автоматически при первом запуске.

### Вопрос: Как добавить новый стек технологий?
**Ответ:** Просто добавьте проекты и разработчиков с новым значением поля `stack` в CSV-файлах. Система автоматически подхватит новый стек при следующем ETL-запуске.
